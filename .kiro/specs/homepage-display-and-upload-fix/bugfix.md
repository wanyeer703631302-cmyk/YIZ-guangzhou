# Bugfix 需求文档

## 介绍

用户报告了网站首页的多个显示和功能问题，包括首屏空白显示、内容位置偏移、画廊显示临时数据而非实际上传的图片，以及上传按钮位置不正确。这些问题严重影响了用户体验和网站的核心功能。本次修复将解决这些显示和数据连接问题，确保首页正确显示内容并连接到 Cloudinary 上的实际图片数据。

## Bug 分析

### 当前行为（缺陷）

1.1 当用户首次访问网站时，首屏显示"暂无内容"的空状态提示，即使后端有数据

1.2 当页面加载完成后，画廊内容需要向下滚动一个完整屏幕高度才能看到

1.3 当画廊显示时，展示的是硬编码的 Unsplash 占位图片（fallbackGalleryData），而不是用户上传到 Cloudinary 的实际图片

1.4 当用户需要上传图片时，上传按钮不在右上角的可见位置，导致用户无法找到上传功能

1.5 当 API 请求 /api/user/interactions 时，服务器返回 500 错误，可能影响数据加载

### 期望行为（正确）

2.1 当用户首次访问网站且后端有数据时，系统应该在首屏直接显示画廊内容，而不是"暂无内容"提示

2.2 当页面加载完成后，系统应该将画廊内容定位在首屏可见区域，无需滚动即可看到

2.3 当画廊显示时，系统应该从 API 获取并显示用户上传到 Cloudinary 的实际图片，只有在 API 失败或无数据时才使用占位图片

2.4 当用户需要上传图片时，系统应该在页面右上角显示上传按钮（与搜索和用户按钮并列），使上传功能易于访问

2.5 当 API 请求 /api/user/interactions 时，系统应该正确处理请求或优雅降级，不应返回 500 错误

### 不变行为（回归预防）

3.1 当 API 确实没有数据时，系统应该继续显示"暂无内容"的空状态提示

3.2 当 API 请求失败时，系统应该继续显示错误状态并提供重试按钮

3.3 当用户点击搜索和用户图标时，系统应该继续正常打开对应的侧边栏

3.4 当用户切换画廊模式（distortion/mouseFollow）时，系统应该继续正常切换显示效果

3.5 当用户选择底部的用户名时，系统应该继续正常切换选中状态

3.6 当画廊有数据时，系统应该继续正常显示加载动画和过渡效果

3.7 当使用 fallbackGalleryData 时，系统应该继续保持其作为后备数据的功能
